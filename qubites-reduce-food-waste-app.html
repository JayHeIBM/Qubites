3 · Raffle System (Leftover / Other types only)

Total Raffles = 3 (base) + pity (accumulated) + 1 (if food matches a preference)
Pity starts at 0. Resets to 0 whenever a user is selected — whether or not they claim the food. Free Food listings never affect pity.

Raffle Rules Summary

Condition Effect on User Pity Change
Food has allergen matching user's listed allergen Excluded from raffle +1 pity
Food does not match user's dietary restriction (e.g., user is vegetarian, food has meat) Excluded from raffle +1 pity
Food matches a user preference (e.g., cuisine type, tag) Eligible + bonus +1 preference raffle
User is selected (offered food) — regardless of whether they claim it Notified, gets access Pity resets to 0
Listing type is Free Food All users get immediate open access No pity changes
Food expires in ≤ 15 minutes (Leftover / Other) All allergen/restriction-safe users get open access No pity changes

5 · Recipient Claiming Flow

  • 1
    Notification received
    Leftover/Other: "You've been selected for Leftover Lasagna – Floor 4. Available until 3:30 PM." Pity resets to 0 now.
    Free Food: "Free Food available – Bagels – Floor 2 Lounge. First-come, first-served."
  • 2
    View listing
    User sees full listing: title, photo, description, confirmed tags, floor, time remaining, and a Claim button.
  • 3
    Go pick up food
    User physically goes to the listed floor/location to retrieve their portion.
  • 4
    Tap "Claimed"
    Serving count decreases by 1. Listing updates in real time. For Leftover/Other, pity was already reset at selection. For Free Food, no pity effect at any point.

6 · Edge Cases & Design Notes

ScenarioBehavior
Fewer eligible users than servings at initial raffle All eligible users are selected. Remaining servings wait for the next 5-min wave.
User selected but never claims (Leftover/Other) After 5 min, their access slot is released back into the pool. Pity remains at 0 (already reset on selection).
Multiple exclusions in same wave (allergen + restriction) Only +1 pity per listing, not +2. Prevents pity inflation from a single food post.
Free Food listing runs out quickly Serving count hits 0. Listing closes. No pity changes for anyone regardless.
Uploader mis-tags allergens Uploader bears responsibility (confirmed via checklist). Report mechanism available to flag a listing.
User profile has no allergens/restrictions set Treated as no restrictions. Eligible for all food. No exclusions, no pity from exclusions.
Free Food listing reaches 15-min expiry window Already open to all — no behaviour change at 15 minutes.
Servings = 0 before expiry timer Listing closes immediately for all types. No further raffle waves.
