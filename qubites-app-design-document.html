Feed & Filtering

  • Default view (no filters active): shows only food the user can access — Free Food listings, raffled Leftover/Other listings the user won, and any listing entering the 15-min open window
  • Filter bar at top allows toggling: Hide my allergens, Abide by my restrictions, Match my preferences
  • Filters are additive and override the default personalised view when active — useful for browsing on behalf of someone else
  • Feed is scrollable and updates in real time as listings are claimed or expire
  • Fully claimed or expired listings are hidden from the feed automatically
  • Free Food listings always appear at the top as they are open to all
The Claim button on a card is only visible when the current user has been selected in the raffle (or the listing is open-access). Other users see only the progress slider.

Profile — Spec

  • Avatar and name pulled from Slack account; not editable in-app
  • Raffle breakdown: Shows Base (always 3), current Pity total, and combined total tickets for the next raffle. If a preference match exists for a currently open listing, a bonus indicator is shown.
  • Edit Allergens / Restrictions / Preferences: Each opens the same checklist UI from onboarding, pre-filled with current selections. Changes take effect on the next raffle wave.
  • Sign Out: Clears session, returns to Login screen. Pity and preferences are persisted server-side.
Pity is visible to the user so they can understand why they may or may not have been selected. It is not gameable — it only accumulates through genuine exclusions.

8 · Upload Flow & AI Tag Verification (Reference)

The Add Page contains the full upload form described in Section 6. The following summarises the AI tag step for reference.

AI Tag Flow

  • On submit of Step 1 details, the title and optional description are sent to the AI model
  • AI returns a list of relevant tag IDs from the master tag list (allergens, restrictions, preferences)
  • These tags are surfaced at the top of each tag group, highlighted in blue, with unchecked checkboxes
  • All other tags in that group are grayed out but remain selectable
  • The uploader must personally check each tag they confirm — AI suggestions are not auto-confirmed
  • Checked tags turn green (confirmed). Unchecked AI suggestions and unselected gray tags are discarded
  • If the copy button was used, all previously confirmed tags are pre-checked and highlighted green — the uploader can uncheck or add tags
  • Submission is blocked until at least one allergen-group review has been performed (even if the result is zero allergens)

9 · Raffle System & Listing Types

Total Raffles = 3 (base) + pity (accumulated) + 1 (if food matches a preference)
Pity starts at 0. Resets to 0 whenever a user is selected — whether or not they claim the food. Free Food listings never affect pity.

11 · Edge Cases & Design Notes

ScenarioBehavior
Fewer eligible users than servings at initial raffleAll eligible users are selected. Remaining servings wait for the next 5-min wave.
User selected but never claims (Leftover/Other)After 5 min, their access slot is released back into the pool. Pity remains at 0 (already reset on selection).
Multiple exclusion reasons on same listingOnly +1 pity per listing. Prevents pity inflation.
Free Food listing runs out quicklyServing count hits 0. Listing closes. No pity changes for anyone.
Copy button used for Add formAll fields pre-populated including tags (pre-checked as confirmed). User can edit any field before submitting as a new listing.
User updates allergens/restrictions mid-dayChanges take effect on the next raffle wave — not retroactively on current open listings.
Uploader mis-tags allergensUploader bears responsibility (confirmed via checklist). A report mechanism is available to flag a listing.
User has no allergens/restrictions setEligible for all food. No exclusions, no pity from exclusions.
Free Food listing at 15-min expiry windowAlready open to all — no behaviour change.
Servings = 0 before expiry timerListing closes immediately for all types. No further raffle waves.
